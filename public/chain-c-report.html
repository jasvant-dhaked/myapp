Executive Summary (Non-Technical)

FamPay checks incoming links to ensure they come from its trusted domain fam.one. However, there is a flaw in how Android parses certain special URL formats. An attacker can create a link that looks like it comes from fam.one to FamPay's security check, but actually opens the attacker's website inside the app.

Think of it like a return address on a letter: the attacker writes "From: fam.one" on the envelope. FamPay checks the return address, sees "fam.one", and opens the letter — but the contents and the real sender are from the attacker.

Once the attacker's page loads inside FamPay, all four JavaScript interfaces are available — granting access to auth tokens, user data, contacts, GPS location, and the ability to fire any internal deeplink.

The Bypass Explained

How Android Parses the URI

Android's Uri.parse() can interpret the "userinfo" part of a URL (before the @ sign) as credentials. In the URI:

https://attacker.com/exploit@fam.one/path

uri.getHost() returns fam.one (what Android considers the host after @)

But a browser/WebView actually navigates to attacker.com/exploit (the authority before @)

Vulnerable Code

// DeepLinkActivity.java line ~2055
String host = data.getHost();
if (host != null && host.equals("fam.one")) {
    // ✅ check passes — but we're actually loading attacker.com!
    startWebViewActivity(data.toString(), ...);
}

Remediation

Fix 1 — Use getAuthority() + Strict Host Parsing (Recommended)

// Replace getHost() with a strict parser that rejects userinfo
private boolean isTrustedHost(Uri uri) {
    // Reject URLs containing @ (userinfo bypass)
    String uriStr = uri.toString();
    if (uriStr.contains("@")) return false;

    String host = uri.getHost();
    return "fam.one".equals(host) || host != null && host.endsWith(".fam.one");
}

Fix 2 — Reject URLs with Userinfo

String rawUrl = data.toString();
if (rawUrl.matches(".*://[^/]*@.*")) {
    // Contains userinfo (@ sign in authority) — reject
    Log.w(TAG, "Blocked URI with userinfo: " + rawUrl);
    return;
}

Fix 3 — URL Normalization before Validation

// Parse with java.net.URL which correctly identifies the host
try {
    URL parsed = new URL(data.toString());
    String host = parsed.getHost(); // correctly returns attacker.com
    if (!"fam.one".equals(host)) return; // blocked
} catch (MalformedURLException e) {
    return; // reject malformed URLs
}
