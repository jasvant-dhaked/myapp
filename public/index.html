CHAIN A WebView JS Interface Exposure HIGH 8.8
Vuln #2 (DeepLink delivery) + Vuln #16 (JS Interface injection)
Opens an attacker URL inside FamPay WebView via a deeplink. The page gains access to four unguarded JavaScript interfaces — capturing auth token, user details, GPS, contacts, and Firebase config.

CHAIN B DAI SSRF HIGH 8.1
Vuln #8 (Internet Service Address Substitution / SSRF) + Vuln #2
Forces FamPay to make HTTP GET/POST requests to any attacker-controlled URL via the /dai/ deeplink handler. Can probe internal API servers, cloud metadata endpoints, and exfiltrate request headers.

CHAIN C HierarchicalUri Domain Bypass HIGH 9.3
Vuln #12 (HierarchicalUri bypass) + Vuln #2 + Vuln #16
Bypasses the fam.one domain allowlist using a malformed URI (https://attacker.com@fam.one). Android's getHost() returns "fam.one" but the WebView loads the attacker's page — with full JS interface access.
🔱 Trigger Exploit Page Report
CHAIN D CropImageActivity + FileProvider File Read HIGH 7.5
Vuln #3 (Exported Activity + Overly Permissive FileProvider)
CropImageActivity is exported with no restrictions. FileProvider exposes the entire app data directory via root-path. Any malicious app can read FamPay's SQLite databases, SharedPreferences (auth tokens), and cached files.

CHAIN E HTML Injection — In-App Phishing CRIT 9.6
Vuln #11 (HTML Injection in Razorpay WebView) + Vuln #2
Injects arbitrary HTML into a WebView loaded with loadDataWithBaseURL("https://api.razorpay.com", ...). Attacker renders a convincing fake payment/UPI PIN screen inside the trusted FamPay app context. Captures credentials.
🎯 Trigger Exploit Page Report
CHAIN F Razorpay OTP Broadcast Interception HIGH 7.8
Vuln #4 (Implicit Broadcast — SmsReceiver)
Razorpay's SmsReceiver sends an implicit broadcast (com.razorpay.events.SMS_PROCESSED) with the full OTP SMS text and sender phone number. Any app on the device can register for this broadcast — no permissions needed.
